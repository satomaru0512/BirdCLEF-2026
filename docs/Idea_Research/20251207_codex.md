了解しました。Sampling_Date をグループ（=土壌・カメラ・天候のセット）として同一 fold にまとめることは絶対条件なので、ストラテジー変更の提案もその制約の上で考えます。今回の EXP060 OOF を分析すると、fold ごとの R² ばらつきは
  「Sampling_Date単位でのデータ分布偏り」と「高さ（Height_Ave_cm）レンジの偏り」に強く依存していました。

  ### ベース実験のズレの本質

  - Fold0: NSW/Vic 2015/1〜9 月の撮影で構成され、Lucerne・Fescue が 68% を占めます。高さ中央値 8.5cm、平均 16cm と他 fold より高背丈。Green/GDM/Total が一貫して過小予測（Green 平均 -12.5g）。Lucerne だけで -30g 近いバイアス。→ “高背丈
    草場 × 特定日程” の固まりを丸ごと扱えていない。
  - Fold1: Tas/WA/Vic の秋撮影で高さ中央値 2.7cm。Clover 占有。Green/GDM が +4〜+9g の過大予測。→ 極低バイオマス圃場だけを含む fold で逆方向バイアス。
  - Fold2/4 は撮影時期・高さ分布が全データ平均に近いため高スコア。Fold3 は Phalaris の高さ要因で fold0 と同じ課題。

  ### 制約下でできる対策

  Sampling_Date をまたがず、未知土壌/Species にも耐える範囲で、以下の改善策が考えられます（train.py 大改造なしで実装可能な順）。

  1. Sampling_Date グループを保ったまま「高さビン」を stratify に追加
      - 例えば Height_Ave_cm を (0–5, 5–15, 15+) にビン分けし、StratifiedGroupKFold の stratify_by を State + height_bin の複合カテゴリにする。
      - これで fold0 のように「高背丈グループが1つの fold に固まる」現象を緩和できます。
      - Sampling_Date 単位で group を守りつつ、fold ごとの高さレンジが近づく。
  2. OOF 残差 × Height の相関を学習時に正則化（高さの比率を使う）
      - Height 自体を loss に入れると fold ごとの差が大きく、今回崩れた。代わりに Green / (Height+ε) のような密度指標を補助 head として学習する。
      - 推論では推定密度 × depth由来のスケールで Green を再構成するので、未知土壌でも「密度」の一般化が効く。Sampling_Date 情報を直接入れないのでリークも無し。
  3. Depth branch をフォールド無依存で事前学習／ウォームアップ
      - fold0 の枚数が少ない状態で depth+height loss を一気にかけると崩れやすい。各 fold の最初3エポックを「backbone全freeze + height loss 0」で回し、depth encoder のみを安定させてから本学習に入る。
      - コード修正は最小で済む。Sampling_Date 制約とは無関係。
  4. Residual補正層を付ける（高さ×depth統計のみ）
      - OOF の Green 残差を f(height, depth_mean, depth_std) で小さな MLP に学習させ、Inference 時も同じ補正を適用する。Species や Sampling_Date を明示的に使わないので未知土壌でも使える。
      - これは “post-hoc calibration” として train.py を大きく変えずに導入可能。
  5. Data augmentation で「疑似スケール変換」を追加
      - Depth と RGB を同時に縦方向に 5〜10% ストレッチ／縮小させる（現実のカメラ高さ変化に近い）Augを加え、モデルに多様な scale を見せる。
      - Sampling_Date 単位で学習／検証を分けても、疑似的な高さ変化を学習できるので fold0 の偏りが緩和される。

  ———

  これらはいずれも Sampling_Date グループを崩さずに導入でき、未知の土壌・Species に対しても「高さやスケールを depth から推測する」準備を整える施策です。特に 1. + 2. を先に試すと、Lucerne/Phalaris vs Clover のような極端な分布差が fold
  ごとに平均化され、R² の底上げにつながるはずです。
