# Fold分割の不均衡問題と解決策

## 概要

本ドキュメントでは、CSIRO Biomass コンペティションにおけるCross Validation（CV）のFold分割問題について、発見の経緯、根拠となるデータ分析、および解決策を記録する。

---

## 1. 問題の発見

### 1.1 発端：OOF予測の誤差分析

EXP024とEXP060のOOF（Out-of-Fold）予測を分析した結果、両モデルで同じ画像が高い誤差を示す傾向が確認された。

**ワースト画像の重複率:**
- Top 20: 16/20 (80%)
- Top 50: 32/50 (64%)

### 1.2 ワースト画像の共通点

誤差が最も大きい画像を調査したところ、明確なパターンが発見された：

## 3. 根本原因の分析

### 3.1 なぜNSWが問題なのか

セクション2の詳細分析から、NSWが他州と大きく異なることが明確になった。

**NSWの特異性（まとめ）:**
- Heightが全体平均の**2.7倍**（20.5cm vs 7.6cm）
- Totalが全体平均の**1.6倍**（70.9g vs 45.3g）
- NSW専用の草種（Fescue, Lucerne, Phalaris）が全75枚中50枚（67%）
- Sampling Dateが6日のみ（他Stateは3-11日）

これらの特異性により、NSWサンプルの学習が不十分だとモデルの予測精度に大きな影響を与える。

### 3.2 Sampling_Dateの構造

データセットのグループ構造を分析：

```
ユニーク Sampling_Date: 28日
画像数: 357枚

注: 一部の日付（例: 2015/6/26）は複数のStateに存在する
- Vic: 2015/6/26 (19枚)
- Tas: 2015/6/26 (18枚)
ただし、異なるStateは異なる場所であり、リークにはならない
```

**State別のSampling_Date数:**
| State | 撮影日数 | 画像数 |
|-------|---------|--------|
| NSW | 6日 | 75枚 |
| Vic | 11日 | 112枚 |
| Tas | 10日 | 138枚 |
| WA | 3日 | 32枚 |

### 3.3 StratifiedGroupKFoldの動作と限界

使用していたCV設定：
```python
StratifiedGroupKFold(n_splits=5, shuffle=True, random_state=42)
# stratify_by='State', group_by='Sampling_Date'
```

**StratifiedGroupKFoldの目的（誤解されやすい点）:**
- ❌ 「各Stateが全Foldに均等に存在する」
- ✅ 「各FoldのValidationセットにおいてStateの**比率**が元データと同じになる」

**制約:**
- 同じSampling_Dateのサンプルは必ず同じFoldに入る
- 各Sampling_DateがState固有なため、完璧なState分散は数学的に不可能

### 3.4 実際のFold分布（問題の可視化）

StratifiedGroupKFoldによる分割結果：

```
State  NSW  Tas  Vic  WA
fold
0       51    0   29   0  ← NSWの68%が集中！
1        0   29   11  12  ← NSW 0%
2        0   26   30   0  ← NSW 0%
3       15   31   14   8
4        9   52   28  12
```

**NSWの6日の割り当て:**
```
Fold 0: 4日 (51枚) ← 2015/1/15, 2015/10/6, 2015/4/1, 2015/5/7
Fold 3: 1日 (15枚) ← 2015/2/24
Fold 4: 1日 (9枚)  ← 2015/2/25
Fold 1, 2: 0日 (0枚)
```

### 3.5 問題の影響

**Fold 0をValidationにした場合:**
- Training = Fold 1 + 2 + 3 + 4
- NSWの訓練データ = 15 + 9 = **24枚** (全75枚中、わずか32%)
- NSWの68%を見ずに学習 → Fold 0のNSW 51枚を予測 → **大失敗**

## 4. 追加の仮説：Test分布

### 4.1 論文の地図分析

論文（HOSTPAPER.md）に記載されたサンプリング地点の地図を分析：

| State | 地図上のサイト数 | Train枚数 | 推測 |
|-------|---------------|----------|------|
| Tas | 2-3箇所（少） | 138枚（39%） | ほぼ全部Trainに使用？ |
| Vic | 8-10箇所（多） | 112枚（31%） | Testに多い？ |
| NSW | 5-6箇所（中） | 75枚（21%） | Testに多い？ |
| WA | 2箇所（少） | 32枚（9%） | 同程度？ |

**仮説:** TestにはVic/NSWが多い可能性が高い

**根拠:**
- 論文記載：全1,162枚（19箇所、4州）
- Train: 357枚、Test: 805枚
- Tasは地図上2-3サイトしかないのに、Trainに138枚もある
- → TasのサイトはほとんどTrainに使われ、TestにはVic/NSWが多い可能性

## 5. 解決策

### 5.1 バランスの取れたFold分割

**目標:** 各FoldのValidation時に、全Stateの75%以上が訓練データに含まれる状態にする

**アルゴリズム:**
1. 各Stateの日付を画像数でソート
2. Greedy法で各日付を「現在最も画像数が少ないFold」に割り当て
3. これにより各State内でFold間の画像数が均等化

### 5.2 実装

スクリプト: `scripts/create_balanced_folds.py`

出力: `input/csiro-biomass/balanced_folds.csv`

### 5.3 改善結果

**最終的なFold分布:**
```
State  NSW  Tas  Vic  WA   合計
fold
0       17   29   25   0    71
1       15   29   25   0    69
2       13   27   21  12    73
3       11   27   21  12    71
4       19   26   20   8    73
```

**Fold枚数のバランス:** 69-73枚（非常に均等）

**Training Coverage比較:**

| Fold | NSW Before | NSW After | 改善 |
|------|-----------|-----------|------|
| 0 | 32% (24枚) | **77%** (58枚) | +45% |
| 1 | 100% | 80% | -20% |
| 2 | 100% | 83% | -17% |
| 3 | 80% | 85% | +5% |
| 4 | 88% | 75% | -13% |

**最小カバレッジ: 32% → 75%** （+43%改善）

### 5.4 追加調整：Fold枚数のバランス改善

初期のGreedy法では以下の問題があった：
- Fold 0, 1にWAが集中 → Fold 0: 101枚, Fold 2: 51枚と不均衡

**調整内容:**
1. **WAの再配置**: Fold 0, 1 → Fold 2, 3, 4 に移動
2. **Tasの再配置**: Tas 2015/6/26 (18枚) を Fold 0 → Fold 2 に移動

これにより Fold枚数が 51-101 → **69-73** に改善された。

### 5.5 設計意図：Stateの優先順位

改善後のFold分布では、NSW/Vic/Tasが全Foldに存在し、WAはFold 2, 3, 4に配置されている。

**優先順位:**
| 優先度 | State | 理由 |
|--------|-------|------|
| 1 | **NSW** | Testに多い可能性 + 元々の学習不足が深刻（Fold 0で32%のみ） |
| 2 | **Vic** | Testに多い可能性（地図上でサイト数が最多） |
| 3 | Tas | Testには少ない可能性（地図上2-3サイトのみ） |
| 4 | WA | 3日しかないため物理的に5 Foldへの完全分散が不可能 |

**Tasがアンバランスでも問題ない理由:**
- Tasは訓練データに十分存在（138枚）
- Fold 2がValidationの場合でも、Tas 129枚（93%）で訓練可能
- Testに少ないと仮説すれば、CVスコアへの影響は限定的
- むしろNSW/Vicのバランスを優先することで、Test予測精度向上が期待できる

**トレードオフの判断:**
- 全Stateを完璧にバランスさせることは、日付の制約上不可能
- 「Testに多いと推測されるStateを優先」という戦略を採用
- 結果としてNSW/Vicは全Foldで75%以上の訓練カバレッジを達成

## 7. リークチェックと詳細分析

### 7.1 リークチェック結果

**結果: リークなし ✓**

同じSampling_Dateが複数のFoldに存在するケースを確認：
- `2015/6/26`: Fold 0 (Vic: 19枚) と Fold 2 (Tas: 18枚)

**これはリークではない理由:**
- 同じ日付でも**異なるState（異なる場所）**であれば問題なし
- GroupKFoldの目的は「同じ場所・同じ条件で撮影された画像」がtrain/valに分かれることを防ぐこと
- Vic (Victoria州) と Tas (Tasmania州) は異なる場所であり、画像の類似性は低い
